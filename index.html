Jira Automation

Data object extraction, manipulation and embedding through Ruby

Jira Cloud Automation and Reporting is a Ruby gem that is used by Optoro's tech teams to add time tracking functionality to Jira Cloud and automate various functions of Optoro's Platform Stability team. The gem uses the Jira API and various other gems (Jiralicous, Business Time) to update Jira issue custom fields with Service Level Agreement and time tracking data. This automation is used to send notifications to product managers, developers, issue watchers, and support personnel to help keep issues on track and to meet contractual SLAs. Ultimately, the data this gem produces is used to create Jira views for tech teams, to analyze tech team performance, and create summary metrics reports for company executives through data mining scripts--all of which informs stability and scaling decisions.

  • Date: November 2016 to Time.now
  • Purpose: fulfill an Optoro business need
  • Tools: Ruby | Rspec | APIs | Jira REST

Global Host

Connecting Travelers

Global Host is a mobile first app that lets travelers connect with guides/hosts in their destination city who share a language and interests, and create itineraries for a day's visit with that host, allowing travelers to experience a city like a local.

Global Host was an eight-day, four-person, final project for Dev Bootcamp. It was built on Rails, utilized the Google Maps JavaScript API, relied on Materialize for visual elements, and was consistently tested using Codeship continuous integration. Check it out on GitHub or try out the App (be sure to search for Chicago once you create a profile, as that's the cit with seed data).

  • Date: July 2016
  • Purpose: DBC Final Project
  • Tools: Ruby | Rails | Rspec | Materialize | Heroku | CI | HTML/CSS/Materialize
